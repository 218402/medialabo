「WEBコンテンツ作成」の成果

本テーマの学習目的

このテーマでは、HTML, CSS, JavaScript の基本を学び、これらを組み合わせて Web ページを構成する方法を学びます。

学習目標:

  • HTML によって Web ページの構造を記述する基本的な方法がわかる。
  • CSS によって Web ページの見た目を設定する基本的な方法がわかる。
  • JavaScript によって Web ページのインタラクティブな処理を記述する基本的な方法がわかる。
  • HTML, CSS, JavaScript の構文リファレンスから、構文の書式や使用方法の情報を入手できる。
  • HTML, CSS, JavaScript, 画像・音声・動画ファイルを組み合わせて簡単な Web ページが作成できる。

なお、Webページのデザインに関する事柄や画像・音声・動画の編集に関連する事柄はこのテーマの範囲外とします。

プロジェクト課題について

　プロジェクト課題ではNHKの番組表を表示するWeb アプリケーションの作成を実施しました。

　チャンネルをプルダウンメニューで選択して、ジャンルの取りうる値を輸入しながら、検索ボタンを押すと、検索結果が「番号」、「番組開始時刻」、「番組終了時刻」、「チャンネル」、「番組名」、「番組サブタイトル」、「番組説明文」、「出演者」をテーブルで表示されます。検索結果のデータ項目を区別やすくするために、情報を番組ごとに新しい表を作成する設計が採用されています。

工夫点，苦心点，強調点，感想

ー・工夫点・ー

　今回のプロジェクト課題で工夫した点は番組チャンネルをプルダウンメニューで選択できるようなデザインで作ることでした。最初はジャンルの取りうる値と同じ入力という方法をとっていましたが、番組チャンネルには数字もアルファベットもあるので入力に不便でしたので、プルダウンメニューに変更しました。

　それからネットページを試行する時に、情報が多すぎるとデータ項目の名前が見えないことに気づきました。例えば開始時間と終了時間、見分けがつかないと大変だと思います。だから検索結果を区別やすくなるために、情報を番組ごとに新しい表を作成する設計が採用されています。

ー・苦心点・ー

　プロジェクト課題を完成させる上で苦心した点はページのデザインについて内容です。第二回の授業にCSSについて知識を勉強して、プロジェクトページの見た目を設定しましたが、授業が進むにつれて、ページの機能が増えてきたので、見た目をを変更する必要がある点が多かったでした。例えばプロジェクト課題の検索結果のデータ項目内容は第二回の授業の時はターブルだけで表示してもいいでしたが、第六回に検索結果を見やすくために削除しました。

　Webページは主にディレクトリとプロジェクト課題で構成されており、2つのインタフェースがより全体的に見えるように、似たようなデザインを採用しています。例えば、両方の背景に文字が見やすい淡い黄色を使って、タイトルのところも目立つ青を使いました。

ー・感　想・ー

　Webページを作成する授業を通じて、HTML、CSS、JavaScriptの知識を学びました。一年生の時、javaを習ったことがありますが、まだ3つの新しい言語利用すると聞き当初は不安を感じました。しかし授業が進むにつれて、HTMLでページを構造して、CSSでページを装飾して、JavaScriptで複雑な機能を実現できました。この過程は何度も試行錯誤していますが、自分のWeb アプリケーションがきれいに機能しているのを見たときも楽しかったです。

　学習の過程で私もよく奇妙な間違いに遭遇しました。例えば第六回の授業でscript要素を追加するのを忘れてエラーが発生しました。先生と先輩はいつも根気よく私を指導してくれて、自分では解決できない多くの難題を解決し助かりました。

　1ヶ月半の間にHTML、CSS、JavaScriptの基礎知識を習得できて、それを利用して自分のWebページを作ったのは本当に不思議なことだと思います。今後はもっときれいで使いやすいページを作るために、Web アプリケーションの作成について知識をもっと勉強したいです。新しいスキルを身につけることは、これからの仕事にも勉強にも有益だと思います。
